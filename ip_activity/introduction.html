Introduction

This interface is used for visualising IP address activity. It contains interactive elements using AJAX technology, so that the page can change dynamically based on user's request without reloading the page.

In order to run properly, the used browser needs JavaScript enabled.
All .html and .css files have been validated by W3C Validator.

Guide

This section contains guide for main operations and offers the description of the main page sections.

Initialisation

The interface is provided by a custom web server, also created for the purposes of this project. Therefore, the activity of the interface is dependent on the server state. In order to interact with the interface, the web server has to be set up and listening on the defined port.

Available Information

The page is divided in two parts. The top part is intended for visualising data of the whole specified address space during the whole time window. The bottom part is hidden until the user selects an area they want to examine further. It contains information about the specified area and the image visualising the area activity is automatically scaled.

The parameters of the stored IP activity is available in Characteristics section. In order to get activity status of a specific cell, the user can hover over the image which will update the coordinates' values in the Current Position section.

Activity Status

Activity status of all IPs can be determined by a distinctive colour at the chosen coordinates. The used colours are defined as follows:

  • black: During the given interval, the IP address was inactive.
  • white: During the given interval, the IP address was active.
  • gray: There is no information about activity for the given interval or address.

Area Selection

The user can select an area for detail examination by choosing both value ranges and submitting the form in the Select Bitmap Area section.

The first way is the manual way. The user can type the selected ranges to the form. It is useful when the user knows the area they want to display separately.

The second way is used for examining the visualised activity based on an interesting pattern found in the image. Instead of having to find out the minimum and maximum values in both dimensions limiting the area, the user can simply drag the cursor and select the area by enclosing it within the displayed frame. The frame size changes based on the cursor direction. When the selection is done, the values of both ranges are automatically inserted in the form.
